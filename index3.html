‚å®Ô∏è What is the Command Prompt?

Your text-based control room

üí°Concepts

  • Command Prompt (CMD) is a text-based interface for Windows
  • Executes commands to perform system actions: open files, copy data, check processes
  • Useful for automation, debugging, and quick system tasks
  • Foundation for Python, R, and Git

üíªBasic Commands

:: Open Command Prompt (Windows + R ‚Üí cmd)
C:\Users\username> ver
:: Displays your Windows version
C:\Users\username> help
:: Lists all available commands
C:\Users\username> exit
:: Closes Command Prompt

üóÇÔ∏è Navigating the File System

Moving through your folders with commands

üí°Key Concepts

  • Current Directory: The folder you're currently "inside"
  • cd ‚Äì change directory
  • dir ‚Äì list contents
  • mkdir ‚Äì make directory
  • cls ‚Äì clear screen
C:\Users\username> dir
:: Lists files and folders
C:\Users\username> cd Desktop
:: Moves into Desktop folder
C:\Users\username\Desktop> mkdir lab_os
:: Creates new folder
C:\Users\username\Desktop> cd lab_os

üìù Creating and Managing Files

Text file operations from the command line

üí°File Commands

  • echo ‚Äì writes text to a file
  • type ‚Äì displays file content
  • del ‚Äì deletes files
  • > ‚Äì redirects output to file (overwrites)
  • >> ‚Äì appends to file
C:\...\lab_os> echo Hello world! > note.txt
:: Creates file with text
C:\...\lab_os> type note.txt
:: Displays content
C:\...\lab_os> echo Line 2 >> note.txt
:: Appends new line
C:\...\lab_os> del note.txt
:: Deletes the file

üì¶ Copying, Moving, and Renaming

Organizing files with precision

üí°File Operations

  • copy ‚Äì duplicates a file
  • move ‚Äì relocates a file
  • rename ‚Äì changes file name
  • Syntax: command source destination
C:\...\lab_os> echo Test > data.txt
C:\...\lab_os> copy data.txt backup.txt
:: Copies file
C:\...\lab_os> move backup.txt C:\Users\username\Desktop
:: Moves file to Desktop
C:\Users\username\Desktop> rename backup.txt results.txt
:: Renames the file

‚ö° Processes and Connectivity

Managing running programs and network

üí°Process & Network Commands

  • tasklist ‚Äì lists running programs
  • taskkill /IM ‚Äì stops a process
  • ping ‚Äì checks internet connection
  • tracert ‚Äì traces route to a site
C:\Users\username> tasklist | find "notepad"
:: Check if Notepad is running
C:\Users\username> taskkill /IM notepad.exe
:: Close Notepad
C:\Users\username> ping google.com
:: Test connection
C:\Users\username> tracert google.com
:: Show network route

üëÅÔ∏è Hidden Files and Attributes

Understanding system file protection

üí°File Attributes

  • Windows uses attributes to protect system files
  • attrib +h ‚Äì hide file
  • attrib -h ‚Äì unhide file
  • Hidden files often store config or cache data
  • Examples: .git, .RData, .ipynb_checkpoints
C:\...\lab_os> echo secret > hidden.txt
C:\...\lab_os> attrib +h hidden.txt
:: Hide file
C:\...\lab_os> dir
:: File not visible
C:\...\lab_os> dir /ah
:: Shows hidden files
C:\...\lab_os> attrib -h hidden.txt
:: Unhide file

ü§ñ Automating with Batch Scripts

Creating your first .bat file

üí°Batch File Concepts

  • Batch files (.bat) store sequences of CMD commands
  • Can be run with a double-click or from terminal
  • Perfect for automating repetitive tasks
  • Foundation for system administration
:: Create a batch script
C:\...\lab_os> echo @echo off > system_check.bat
C:\...\lab_os> echo echo Running check... >> system_check.bat
C:\...\lab_os> echo systeminfo ^> sys_report.txt >> system_check.bat
C:\...\lab_os> echo ping google.com >> system_check.bat
C:\...\lab_os> echo pause >> system_check.bat
C:\...\lab_os> system_check.bat
:: Runs the script

‚ö° Introduction to PowerShell

The next generation command line

üí°What is PowerShell?

  • Advanced command-line interface created by Microsoft
  • Works with objects, not just text
  • Built on .NET framework
  • Uses cmdlets with Verb-Noun syntax (Get-Process, Stop-Service)
  • Ideal for automation and complex system tasks
# Open PowerShell (Windows + X ‚Üí Windows PowerShell)
PS C:\Users\username> Get-Host
# Displays version info
PS C:\Users\username> Get-Help
# Lists available commands
PS C:\Users\username> Get-Command
# Shows all cmdlets

üìÇ PowerShell File Management

Working with files and folders

üí°Key Cmdlets

  • Get-ChildItem ‚Äì list files/folders
  • Set-Location ‚Äì change directory
  • New-Item ‚Äì create files/folders
  • Remove-Item ‚Äì delete items
PS C:\Users\username> Get-ChildItem
# Lists files (like 'dir')
PS C:\Users\username> Set-Location Desktop
PS C:\...\Desktop> New-Item -Name "ps_lab" -ItemType Directory
# Creates folder
PS C:\...\Desktop> Set-Location ps_lab
PS C:\...\ps_lab> New-Item -Name "notes.txt" -ItemType File

üìù PowerShell File Content

Reading and writing files

üí°Content Cmdlets

  • Set-Content ‚Äì write new content (overwrites)
  • Add-Content ‚Äì append text
  • Get-Content ‚Äì read file content
  • PowerShell treats files as structured objects
PS C:\...\ps_lab> Set-Content -Path notes.txt -Value "PowerShell is powerful!"
# Writes to file
PS C:\...\ps_lab> Add-Content -Path notes.txt -Value "It automates tasks!"
# Appends text
PS C:\...\ps_lab> Get-Content -Path notes.txt
# Displays content

‚öôÔ∏è PowerShell System Management

Monitoring and controlling processes

üí°Process Management

  • Get-Process ‚Äì list running processes
  • Stop-Process ‚Äì terminate a process
  • Get-ComputerInfo ‚Äì system details
  • Use pipes (|) to chain commands and filter data
PS C:\Users\username> Get-Process
# Lists all processes
PS C:\Users\username> Get-Process | Where-Object { $_.CPU -gt 10 }
# Filters high CPU usage
PS C:\Users\username> Stop-Process -Name "notepad"
PS C:\Users\username> Get-ComputerInfo

üìú PowerShell Scripts

Creating .ps1 automation files

üí°Script Concepts

  • PowerShell scripts use .ps1 extension
  • More powerful than batch files
  • May need to set execution policy: Set-ExecutionPolicy RemoteSigned
  • Perfect for automated workflows
PS C:\...\ps_lab> @'
Write-Output "Running system check..."
Get-ComputerInfo | Out-File "sys_report.txt"
Write-Output "Network test:"
Test-Connection google.com -Count 2
Write-Output "Done!"
'@ > system_check.ps1
PS C:\...\ps_lab> .\system_check.ps1

üé§ Speaker Notes

PowerShell takes what CMD can do ‚Äî and goes much further.

It's object-oriented: every output is an object, not just plain text. That means you can extract, filter, and manipulate data directly.

Each PowerShell command is called a cmdlet and follows the syntax Verb-Noun (like Get-Process, Stop-Service, etc.).

×

üí° Code Explanation

Get-Help - Displays help system and available cmdlets

Notice the Verb-Noun pattern? This makes commands self-explanatory.

üé§ Speaker Notes

This makes scripts easier to read ‚Äî no abbreviations, just clear instructions.

üí° Code Explanation

PowerShell is more verbose but also more explicit and powerful!

üí° Code Explanation
