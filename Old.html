Why Programming for Psychologists?

Programming skills enable you to:

  • Analyze research data efficiently
  • Create psychological experiments and surveys
  • Develop VR applications for therapy
  • Automate repetitive tasks in research
  • Visualize complex psychological data
  • Understand modern cognitive science tools
  • Learn other languages and tools more easily (Inquisit, R, SPSS scripting, etc.)
  • Enhance your career opportunities in research and industry

Why Python?

  • Most popular language in data science and research
  • Readable and beginner-friendly syntax
  • Extensive libraries for psychology research (PsychoPy, pandas, etc.)
  • Large supportive community
  • Industry-standard for data analysis

Course Structure - What We'll Learn

Theoretical Knowledge (Course - Prof. Fischmann):

  • Computer architecture and hardware
  • Operating systems and file systems
  • Data types and code structure
  • Control flow (conditionals, loops)
  • Functions and code reuse
  • Exception handling
  • Testing and libraries

Lab Structure - What We'll Practice

  • Weeks 2-3: Computer hardware and operating systems
  • Week 4: GitHub, Codespaces, first program
  • Week 5: Python data types and F-strings
  • Week 6: Conditionals (if, elif, else)
  • Week 7: Functions and parameters
  • Week 8: Loops (for, while, break, continue)
  • Week 9: Exception handling

Lab Structure - Continued

  • Week 10: Testing in Python
  • Week 11: Working with libraries - Project begins!
  • Weeks 12-13: Data visualization and project work
    • You'll create your own Python program
    • Apply everything you've learned
    • Present your work

Evaluation - Course Component (50%)

Multiple Choice Exam:

  • Total: 36 questions
  • Minimum to pass: 16 correct answers (44%)
  • Grading: Starts from 1
    • +0.25 points per correct answer
    • No points deducted for wrong answers
  • Maximum grade: 10

Evaluation - Lab Component (50%)

End-of-Term Python Project:

Minimum to pass: 4 out of 9 points

  • Program runs without errors (1 point)
  • Program produces expected results (1 point)
  • Includes meaningful test (1 point)
  • Live modification during presentation (3 points)
    • 5 minutes to implement requested change
  • Oral defense - answer questions about your code (3 points)
    • 1 minute per question

Evaluation - Important Notes

Both components must be passed separately
  • Failing either = failing the course
  • Course must be retaken next year
  • Final grade = arithmetic mean of course and lab
Zero tolerance for plagiarism
  • Results in minimum grade regardless of work quality
  • Your code, your understanding, your success
